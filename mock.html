✅ Step-by-step:

The first 10 results are: results[0] to results[9]

To reverse them, we must start from results[9] and go backward to results[0].

🔍 Understanding the Slice:

results[9::-1]

  • 9 = start index (10th item)
  • ::-1 = step of -1 (reverse)
  • No explicit stop index → it stops at the beginning (index 0)

    ❌ Returns an empty list because slicing in reverse needs start > stop.
  • D) results[::-1][:10]
    Reverses the entire list first (all 1000 items), then takes the first 10 of that reversed list.

# Class vs Instance Variable Example
class Car:
    wheels = 4  # class variable (shared)

    def __init__(self, brand):
        self.brand = brand  # instance variable

c1 = Car("BMW")
c2 = Car("Tesla")
c2.wheels = 6  # creates instance variable for c2

print(c1.wheels)  # Output: 4
print(c2.wheels)  # Output: 6

🧠 Explanation:

🔍 Step-by-Step Breakdown:

  1. Class Car is defined with:
    • Class variable wheels = 4 (shared by all instances)
    • Instance variable brand (unique to each car)
  2. Two instances are created:
    • c1 = Car("BMW")
    • c2 = Car("Tesla")
  3. The critical line: c2.wheels = 6
    • Creates a new instance variable wheels just for c2
    • Does not modify the class variable

🧪 Output Analysis:

Object wheels Value Explanation
c1 4 Uses class variable since no instance variable exists
c2 6 Uses its own instance variable that shadows the class variable

💡 Key Concepts:

  • Class variables are shared across all instances
  • Instance variables are specific to each object
  • Assignment to an instance creates a new variable that shadows the class variable
  • Original class variable remains unchanged and accessible through other instances

🔧 Checking Variable Status:

print(Car.wheels)  # 4 (class variable unchanged)
print(c1.__class__.wheels)  # 4 (accessing class variable)
print(c2.__dict__)  # {'brand': 'Tesla', 'wheels': 6} (instance variables)
